## 2025-05-14 - [Contact Form Accessibility & Shadcn Linting]
**Learning:** For accessible forms where only placeholders are desired, use `<Label>` components with the `sr-only` class to provide context to screen readers without changing the visual design. When using `aria-describedby` with third-party error components like `ValidationError`, wrap the component in a `div` with the target `id` to ensure the link works regardless of the component's internal prop forwarding. Also, Shadcn components may trigger `@typescript-eslint/no-empty-object-type` if interfaces are empty; add explicit members like `children` or `className` to resolve.
**Action:** Always implement `sr-only` labels for placeholder-only inputs and use a wrapper `div` for `aria-describedby` targets when using external error components. Ensure Shadcn interfaces have at least one member to pass ESLint.
